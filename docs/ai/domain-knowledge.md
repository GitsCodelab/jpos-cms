# Domain Knowledge (Fintech CMS)

Last updated: 2026-05-03

## Scope

This platform supports operations teams for card/payment ecosystems.
Core domains:
- reconciliation
- settlement
- fraud monitoring
- operational administration

## Key Concepts

### Transaction
A payment event imported from external systems with amount, currency, direction, status, and references.

### Reconciliation
Process of matching transaction records across sources to find matched, unmatched, and exceptions.

### Settlement
Financial netting and batching process that determines debit/credit positions and formal statements.

### Fraud Alert
Risk signal generated by rule evaluation, tracked through review and resolution statuses.

### Operational Rules
Configurable behavior policies (accounting, authorization, processing, checks, disputes).

## Roles (Current and target)

Current live role usage:
- `admin` (seeded default)

Target model (planned expansion):
- admin
- operator
- viewer
- specialized operational roles

## Data Reality

- CMS reads and manages operational data.
- Switch-origin message processing belongs to the external switch project.
- CMS should remain Oracle-ready even if local development uses SQLite.
